39  /**
40  * Query result returned by `query_once` and `query`.
41  *
42  * 'wait_input' indicates whether a selection is confirmed, either by
43  * "CONFIRM" action or by the option's corresponding action, or if the
44  * popup is canceled by "QUIT" action when `allow_cancel` is set to true.
45  * It is also false if `allow_anykey` is set to true, or when an error
46  * happened. It is always false when returned by `query`.
47  *
48  * `action` is the selected action, "QUIT" if `allow_cancel` is set to
49  * true and "QUIT" action occurs, or "ANY_INPUT" if `allow_anykey` is
50  * set to true and an unknown action occurs. In `query_once`, action
51  * can also be other actions such as "LEFT" or "RIGHT" which are used
52  * for moving the cursor. If an error occurred, such as when the popup
53  * is not properly set up, `action` will be "ERROR".
54  *
55  * `evt` is the actual `input_event` that triggers the action. Note that
56  * `action` and `evt` do NOT always correspond to each other. For
57  * example, if an action is selected by pressing return ("CONFIRM" action),
58  * `action` will be the selected action, while `evt` will correspond to

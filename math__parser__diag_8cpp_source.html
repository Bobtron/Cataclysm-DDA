23 /*
24 General guidelines for writing dialogue functions
25 
26 The typical parsing function takes the form:
27 
28 std::function<double( dialogue & )> myfunction_eval( char scope,
29  std::vector<diag_value> const &params, diag_kwargs const &kwargs )
30 {
31  diag_value myval( std::string{} );
32  if( kwargs.count( "mykwarg" ) != 0 ) {
33  myval = *kwargs.at( "mykwarg" );
34  }
35 
36  ...parse-time code...
37 
38  return[effect_id = params[0], myval, beta = is_beta( scope )]( dialogue const & d ) {
39  ...run-time code...
40  };
41 }
42 
43 - Don't validate the number of arguments (params). The math parser already does that
44 - Only use variadic functions if all arguments are treated the same way,
45  regardless of how many there are (including zero)
46 - Use kwargs for optional arguments
47 - Prefer splitting functions instead of using mandatory kwargs
48  ex: school_level() split from spell_level() instead of spell_level('school':blorg)
49 - Use parameter-less functions diag_value::str(), dbl(), and var() only at parse-time
50 - Use conversion functions diag_value::str( d ) and dbl( d ) only at run-time
51 - Always throw on errors at parse-time
52 - Never throw at run-time. Use a debugmsg() and recover gracefully
53 */

◆ is_null()

template<typename T >
bool string_id< T >::is_null ( ) const
inline

Returns whether this represents the id of the null-object (in which case it's the null-id).

Note that not all types assigned to T may have a null-object. As such, there won't be a definition of NULL_ID and if you use any of the related functions, you'll get errors during the linking.

Example: "mon_null" is the id of the null-object of monster type.

Note: per definition the null-id shall be valid. This allows to use it in places that require a (valid) id, but it can still represent a "don't use it" value.

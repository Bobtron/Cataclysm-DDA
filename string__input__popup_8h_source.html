22 /**
23  * Shows a window querying the user for input.
24  *
25  * Returns the input that was entered. If the user cancels the input (e.g. by pressing Escape),
26  * an empty string is returned. An empty string may also be returned when the user does not enter
27  * any text and confirms the input (by pressing ENTER).
28  *
29  * Examples:
30  * \code
31  input = string_input_popup().title("Enter something").query();
32  // shows the input field in window w at coordinates (10,20) up to (30,20).
33  input = string_input_popup().window(w, 10, 20, 30).query();
34  * \endcode
35  *
36  * @param title The displayed title, describing what to enter. @ref color_tags can be used.
37  * @param width Width of the input area where the user input appears.
38  * @param input The initially display input. The user can change this.
39  * @param desc An optional text (e.h. help or formatting information) which is displayed
40  * above the input. Color tags can be used.
41  * @param identifier If not empty, this is used to store and retrieve previously entered
42  * text. All calls with the same `identifier` share this history, the history is also stored
43  * when saving the game (see @ref uistate).
44  * @param max_length The maximal length of the text the user can input. More input is simply
45  * ignored and the returned string is never longer than this.
46  * @param only_digits Whether to only allow digits in the string.
47  */

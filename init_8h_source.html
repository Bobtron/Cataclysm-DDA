22 /**
23  * This class is used to load (and unload) the dynamic
24  * (and moddable) data from json files.
25  * There exists only one instance of this class, which
26  * can be accessed with @ref get_instance
27  *
28  * Usage is basically this:
29  * - Let user decide which world to play in.
30  * - Call @ref unload_data (to unload data from a
31  * previously loaded world, if any)
32  * - Call @ref load_data_from_path(...) repeatedly with
33  * different paths for the core data and all the mods
34  * of the current world.
35  * - Call @ref finalize_loaded_data when all mods have been
36  * loaded.
37  * - Play.
38  *
39  * The object initializes itself upon first usage.
40  * It also unloads everything when the program ends.
41  *
42  *
43  *
44  * Porting stuff to json works like this:
45  * - create a function
46  * void load_my_object(JsonObject &jo);
47  * - Add an entry to @ref type_function_map (inside of @ref initialize)
48  * that calls the new function.
49  * - Inside that function load the data from the json object.
50  * You must also provide a reset function and add a call to
51  * that function in @ref unload_data
52  * - Optional: create a finalize function and call it from
53  * @ref finalize_loaded_data
54  * - Optional: create a function to check the consistency of
55  * the loaded data and call this function from @ref check_consistency
56  * - Than create json files.
57  */

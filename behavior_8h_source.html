29 // The behavior tree is (at least initially) intended to decide on a goal for a given subject.
30 // Each time tick is called, it evaluates the tree to find the highest-priority behavior that is
31 // capable of making forward progress.
32 // The behavior tree performs a depth-first traversal until it reaches an unmet goal.
33 // When tick is invoked, it visits root -> node_t::tick -> strategy -> children -> node_t::tick
34 // At each level, the strategy determines the order in which to visit the current node's children.
35 // Once a leaf node is reached, it either returns success, indicating its requirements are met,
36 // failure, indicating that it is incapable of satisfying its requirements, or running, indicating
37 // that it is capable of addressing its requirements, but that they aren't met yet.
38 // In practice, the tree is traversed until it reaches a node that returns running,
39 // meaning that it is capable of making progress if set as a goal.
40 // The arrangement of the tree and configuration of iteration strategies guarantees that it visits
41 // nodes in order of descending priority.
42 // A predicate can be injected into any node to direct iteration,
43 // including skipping entire subtrees.

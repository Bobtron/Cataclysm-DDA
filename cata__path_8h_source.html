9 /**
10  * One of the problems of filesystem paths is they lack contextual awareness
11  * of 'where' they are. They're just a list of segments pointing into the
12  * filesystem. There are cases where it's useful to have a path relative to
13  * some logical root which isn't the actual filesystem root, and work with /
14  * manipulate that path independently of the logical root.
15  *
16  * cata_path combines a logical root concept and an fs::path. The logical
17  * root is an enum value containing well known cataclysm 'roots'. It only
18  * exposes operator / for concatenating segments to the relative path.
19  * The constructor requires passing a logical root value which can be queried,
20  * and when (explicitly) converting to fs::path will prepend the logical root
21  * with the relative path to give a final absolute path. Callers can know this
22  * absolute path is relative to the logical root without having to perform any
23  * further tests.
24  *
25  * Attempting to construct a path that 'escapes' the logical root will result
26  * in a path rooted at the logical root, just like what happens if you try to
27  * resolve '/../'.
28  *
29  * All string values are assumed to be utf-8 encoded, but fs::path values are
30  * used as-is without checking for encoding, so fs::u8path should be used if the
31  * path is utf-8 encoded.
32  */

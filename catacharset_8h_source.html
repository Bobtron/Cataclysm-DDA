65 /**
66  * UTF8-Wrapper over std::string.
67  * It looks and feels like a std::string, but uses code points counts
68  * as index, not bytes.
69  * A multi-byte Unicode character might be represented
70  * as 3 bytes in UTF8, this class will see these 3 bytes as 1 character.
71  * It will never separate them. It will however split between code points
72  * which might be problematic when containing combination characters.
73  * In this case use the *_display functions. They operate on the display width.
74  * Code points with a zero width are considered to belong to the previous code
75  * point and are not split from that.
76  * Having a string with like [letter0][letter1][combination-mark][letter2]
77  * (assuming each letter has a width of 1) will return a display_width of 3.
78  * substr_display(1, 2) returns [letter1][combination-mark][letter2],
79  * substr_display(1, 1) returns [letter1][combination-mark]
80  * substr_display(2, 1) returns [letter2]
81  *
82  * Note: functions use code points, not bytes, for counting/indexing!
83  * Functions with the _display suffix use display width for counting/indexing!
84  * Protected functions might behave different
85  *
86  * For function documentation see std::string, the functions here
87  * mimic the behavior of the equally named std::string function.
88  */

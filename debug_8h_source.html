8 /**
9  * debugmsg(msg, ...)
10  * varg-style functions: the first argument is the format (see printf),
11  * the other optional arguments must match that format.
12  * The message is formatted and printed on screen to player, they must
13  * acknowledge the message by pressing SPACE. This can be quite annoying, so
14  * avoid this function if possible.
15  * The message is also logged with DebugLog, a separate call to DebugLog
16  * is not needed.
17  *
18  * DebugLog
19  * The main debugging system. It uses debug levels (how critical/important
20  * the message is) and debug classes (which part of the code it comes from,
21  * e.g. mapgen, SDL, npcs). This allows disabling debug classes and levels
22  * (e.g. only write SDL messages, but no npc or map related ones).
23  * It returns a reference to an outputs stream. Simply write your message into
24  * that stream (using the << operators).
25  * DebugLog always returns a stream that starts on a new line. Don't add a
26  * newline at the end of your debug message.
27  * If the specific debug level or class have been disabled, the message is
28  * actually discarded, otherwise it is written to a log file.
29  * If a single source file contains mostly messages for the same debug class
30  * (e.g. mapgen.cpp), create and use the macro dbg.
31  *
32  * dbg
33  * Usually a single source contains only debug messages for a single debug class
34  * (e.g. mapgen.cpp contains only messages for D_MAP_GEN, npcmove.cpp only D_NPC).
35  * Those files contain a macro at top:
36 @code
37 #define dbg(x) DebugLog((x), D_NPC) << __FILE__ << ":" << __LINE__ << ": "
38 @endcode
39  * It allows to call the debug system and just supply the debug level, the debug
40  * class is automatically inserted as it is the same for the whole file. Also this
41  * adds the file name and the line of the statement to the debug message.
42  * This can be replicated in any source file, just copy the above macro and change
43  * D_NPC to the debug class to use. Don't add this macro to a header file
44  * as the debug class is source file specific.
45  * As dbg calls DebugLog, it returns the stream, its usage is the same.
46  */

27 /**
28 A generic class to store objects identified by a `string_id`.
29 
30 The class handles loading (including overriding / replacing existing objects) and
31 querying for specific objects. The class is designed to work with @ref string_id and
32 can be by it to implement its interface.
33 
34 ----
35 
36 @tparam T The type of the managed objects. The type must have:
37  - a default constructor,
38  - a `load( JsonObject & )` function,
39  - an `id` member of type `string_id<T>`,
40  - a `was_loaded` member of type `bool`, which must have the value `false` before
41  the first call to `load`.
42 
43  The type can also have:
44  - a 'check()' function (to run `generic_factory::check()` on all objects)
45 
46  Those things must be visible from the factory, you may have to add this class as
47  friend if necessary.
48 
49  `T::load` should load all the members of `T`, except `id` and `was_loaded` (they are
50  set by the `generic_factory` before calling `load`). Failures should be reported by
51  throwing an exception (e.g. via `JsonObject::throw_error`).
52 
53 ----
54 
55 Usage:
56 
57 - Create an instance, it can be static, or packed into another object.
58 - Implement `string_id::load` as simply forwarding to `generic_factory::load`.
59  Register `string_id::load` in the @ref DynamicDataLoader (init.cpp) to be called when
60  an object of the matching type is encountered.
61 - Similar: implement and register `string_id::reset` and let it call `generic_factory::reset`.
62 
63 The functions can contain more code:
64 - `load` typically does nothing special beside forwarding to `generic_factory`.
65 - `reset` removes the loaded objects. It usually needs to remove the additional data that was set
66  up in `finalize`. It must call `generic_factory::reset`.
67 
68 Optional: implement the other functions used by the DynamicDataLoader: `finalize`,
69 `check_consistency`. There is no implementation of them in the generic factory.
70 
71 `check_consistency` typically goes over all loaded items and checks them somehow.
72 
73 `finalize` typically populates some other data (e.g. some cache) or sets up connection between
74 loaded objects of different type.

558 /**
559 @file
560 Helper for loading from JSON
561 
562 Loading (inside a `T::load(JsonObject &jo)` function) can be done with two functions
563 (defined here):
564 - `mandatory` loads required data and throws an error if the JSON data does not contain
565  the required data.
566 - `optional` is for optional data, it has the same parameters and an additional default
567  value that will be used if the JSON data does not contain the requested data. It may
568  throw an error if the existing data is not valid (e.g. string instead of requested int).
569 
570 The functions are designed to work with the `generic_factory` and therefore support the
571 `was_loaded` parameter (set by `generic_factory::load`). If that parameter is `true`, it
572 is assumed the object has already been loaded and missing JSON data is simply ignored
573 (the default value is not applied and no error is thrown upon missing mandatory data).
574 
575 The parameters are this:
576 - `JsonObject jo` the object to load from.
577 - `bool was_loaded` whether the object had already been loaded completely.
578 - `std::string member_name` the name of the JSON member to load from.
579 - `T &member` a reference to the C++ object to store the loaded data.
580 - (for `optional`) a default value of any type that can be assigned to `member`.
581 
582 Both functions use the native `read` functions of `JsonIn` (see there) to load the value.
583 
584 Example:
585 \code
586 class Dummy {
587  bool was_loaded = false;
588  int a;
589  std::string b;
590  void load(JsonObject &jo) {
591  mandatory(jo, was_loaded, "a", a);
592  optional(jo, was_loaded, "b", b, "default value of b");
593  }
594 };
595 \endcode
596 
597 This only works if there is function with the matching type defined in `JsonIn`. For other
598 types, or if the loaded value needs to be converted (e.g. to `nc_color`), one can use the
599 reader classes/functions. `mandatory` and `optional` have an overload that requires the same
600 parameters and an additional reference to such a reader object.
601 
602 \code
603 class Dummy2 {
604  bool was_loaded = false;
605  int b;
606  nc_color c;
607  void load(JsonObject &jo) {
608  mandatory(jo, was_loaded, "b", b); // uses JsonIn::read(int&)
609  }
610 };
611 \endcode
612 
613 Both versions of `optional` have yet another overload that does not require an explicit default
614 value, a default initialized object of the member type will be used instead.
615 
616 ----
617 
618 Readers must provide the following function:
619 `bool operator()( const JsonObject &jo, const std::string &member_name, T &member, bool was_loaded ) const
620 
621 (This can be implemented as free function or as operator in a class.)
622 
623 The parameters are the same as the for the `mandatory` function (see above). The `function shall
624 return `true` if the loading was done, or `false` if the JSON data did
625 not contain the requested member. If loading fails because of invalid data (but not missing
626 data), it should throw.
627 
628 */

977 /**
978  * Base class for reading generic objects from JSON.
979  * It can load members being certain containers or being a single value.
980  * The function get_next() needs to be implemented to read and convert the data from JSON.
981  * It uses the curiously recurring template pattern, you have to derive your new class
982  * `MyReader` from `generic_typed_reader<MyReader>` and implement `get_next` and
983  * optionally `erase_next`.
984  * Most function calls here are done on a `Derived`, which means it can "override" them.
985  * This even allows changing their signature and return type.
986  *
987  * - If the object is new (`was_loaded` is `false`), only the given JSON member is read
988  * and assigned, overriding any existing content of it.
989  * - If the object is not new and the member exists, it is read and assigned as well.
990  * - If the object is not new and the member does not exists, two further members are examined:
991  * entries from `"extend"` are added to the set and entries from `"delete"`
992  * are removed. This only works if the member is actually a container, not just a single value.
993  *
994  * Example:
995  * The JSON `{ "f": ["a","b","c"] }` would be loaded as the set `{"a","b","c"}`.
996  * Loading the set again from the JSON `{ "delete": { "f": ["c","x"] }, "extend": { "f": ["h"] } }`
997  * would add the "h" flag and removes the "c" and the "x" flag, resulting in `{"a","b","h"}`.
998  *
999  * @tparam Derived The class that inherits from this. It must implement the following:
1000  * - `Foo get_next( V ) const`: reads the next value from JSON, converts it into some
1001  * type `Foo` and returns it. The returned value is assigned to the loaded member (see reader
1002  * interface above), or is inserted into the member (if it's a container). The type `Foo` must
1003  * be compatible with those uses (read: it should be the same type). V is a type convertible from
1004  * JsonValue.
1005  * - (optional) `erase_next( V v, C &container ) const`, the default implementation here
1006  * reads a value from JSON via `get_next` and removes the matching value in the container.
1007  * The value in the container must match *exactly*. You may override this function to allow
1008  * a different matching algorithm, e.g. reading a simple id from JSON and remove entries with
1009  * the same id from the container. V is a value convertible from JsonValue.
1010  */
1011 template<typename Derived>
1012 class generic_typed_reader
1013 {
1014 public:
1015  template<typename C>
1016  void insert_values_from( const JsonObject &jo, const std::string_view member_name,
1017  C &container ) const {
1018  const Derived &derived = static_cast<const Derived &>( *this );
1019  if( !jo.has_member( member_name ) ) {
1020  return;
1021  }
1022  JsonValue jv = jo.get_member( member_name );
1023  // We allow either a single value or an array of values. Note that this will not work
1024  // correctly if the thing we load from JSON is itself an array.
1025  if( jv.test_array() ) {
1026  for( JsonValue jav : jv.get_array() ) {
1027  derived.insert_next( jav, container );
1028  }
1029  } else {
1030  derived.insert_next( jv, container );
1031  }
1032  }
1033 
1034  template<typename C>
1035  void insert_next( JsonValue &jv, C &container ) const {
1036  const Derived &derived = static_cast<const Derived &>( *this );
1037  reader_detail::handler<C>().insert( container, derived.get_next( jv ) );
1038  }
1039 
1040  template<typename C>
1041  void erase_values_from( const JsonObject &jo, const std::string_view member_name,
1042  C &container ) const {
1043  const Derived &derived = static_cast<const Derived &>( *this );
1044  if( !jo.has_member( member_name ) ) {
1045  return;
1046  }
1047  JsonValue jv = jo.get_member( member_name );
1048  // Same as for inserting: either an array or a single value, same caveat applies.
1049  if( jv.test_array() ) {

1051  derived.erase_next( jav, container );

1053  } else {
1054  derived.erase_next( jv, container );
1055  }
1056  }

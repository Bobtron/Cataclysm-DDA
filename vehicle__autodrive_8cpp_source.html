39 /*
40  * This file contains code that allows a vehicle to be driven by an in-game character (most
41  * likely the player character) to some remote destination, following a path of adjacent overmap
42  * tile (OMT) coordinates, while avoiding any obstacles along the way.
43  *
44  * The main entry point is do_autodrive(), which is intended to be called from the do_turn() of
45  * a long-running activity. This will cause the driver character to perform one or more driving
46  * actions (as long as they have enough moves), which could be steering and/or changing the
47  * desired speed. The driver may also do nothing if the vehicle is going in the right
48  * direction.
49  *
50  * Most of the logic is inside the private implementation class autodrive_controller, which is
51  * instantiated at the beginning of an autodrive activity and cleaned up at the end. This class
52  * caches information about the vehicle, driver and the local terrain in order to minimize the
53  * amount of computation required on each invocation of do_autodrive().
54  *
55  * Upon reaching a new overmap tile, as well as once initially, the autodrive algorithm will
56  * try to find a path that takes the vehicle from its current position through the current
57  * and next OMT in order to reach the 2nd next OMT in the overall path to destination (which
58  * is obtained from driver.omt_path). Thus we always plan ahead 2 OMTs worth of navigation,
59  * but we only follow the first half of the route, then recompute a new route for the next
60  * 2 OMTs. This ensures that we don't drive up to the edge of an OMT only to get stuck due
61  * to a large obstacle that can't be driven around from that position (a dead end) and instead
62  * we always choose to cross OMT boundaries in a position from which we are guaranteed to be
63  * able to continue onward.
64  *
65  * The pathfinding algorithm being employed is A*. The graph being searched is made up of nodes
66  * denoted by x and y coordinates as well as a direction of travel. Since steering changes the
67  * vehicle's direction in increments of 15 degrees there are 360/15 possible orientations. By
68  * limiting navigation to 2 OMTs at a time we have a maximum search space of 2*24*24*24 nodes.
69  * In practice, A* finds a good path after visiting fewer than 50 nodes due to the heuristic
70  * function that estimates which nodes are most likely to lead to the destination (which is
71  * based on proximity to the "exit zone" leading to the next OMT and whether the vehicle is
72  * pointing towards the middle of that zone). Although there can be many paths that we could
73  * take to get through to the next OMT, we just take the first one we find since it tends to
74  * be the most direct and is usually near-optimal. When the way is blocked we'll either give
75  * up after exhausting the search space or after we hit a limit on the search length.
76  *
77  * In order to build the search graph for A*, we consider every possible map tile and orientation
78  * in the search zone and we figure out whether the vehicle's pivot point could be placed there
79  * without the vehicle overlapping some obstacle (a collision-free placement). The pivot point
80  * is the part of the vehicle that moves the least while steering and is usually between the
81  * wheels of the vehicle, closer to the non-steerable ones. The pathfinding problem reduces to
82  * finding a path for the pivot point to follow through the set of possible positions and
83  * orientations (nodes). In order to figure out the possible connections between nodes (the
84  * edges of the graph) we consider a limited set of actions that the driver can perform at
85  * each location: nothing, steer left or steer right (possibly more than once). Given a current
86  * node, current speed and amount of steering we can predict where the vehicle's pivot point
87  * will end up in 1 turn and connect the 2 nodes.
88  *
89  * In order to keep the search space small we only consider paths traveled at constant speed
90  * (otherwise we'd have to add speed as an extra attribute in every node). When we need to
91  * compute a new path we first try to use the maximum safe autodrive speed (which is typically
92  * much less than the vehicle's max speed). If we can't find a path we cut the speed in half and
93  * try again, repeating until we succeed or reach the minimum speed. When the target speed is
94  * different from the current speed of the vehicle we try to estimate how long it will take to
95  * reach that speed and take that into account when simulating the first few edges (they'll be
96  * shorter or longer than the rest).
97  *
98  * Once we have generated a sequence of nodes to follow, we do not invoke A* again until we reach
99  * the next OMT. We simply cache the path and on every subsequent call to do_autodrive() we
100  * check where we are on that path, what node we're supposed to reach next and what action we
101  * should use to get there. However, sometimes the simplified simulation used to compute the
102  * search graph fails to accurately predict where the vehicle will end up. This could be because
103  * there was too much error in an approximation (such as time to accelerate) or the driving
104  * conditions changed (ran over a mound of dirt that interfered with steering). When such a
105  * discrepancy is detected, autodrive will fall back to "safe mode", cutting the speed to the
106  * minimum (1 tile per second) and recomputing the path.
107  *
108  * Since the navigation graph is cached and not updated in response to dynamic obstacles (such
109  * as animals) autodrive will also perform collision detection at every turn, before taking
110  * any action that may end the turn. If a possible collision is detected autodrive will enter
111  * "safe mode", reducing speed and recomputing. If the speed is already at minimum it will
112  * abort instead with a warning message.
113  *
114  * In most cases when autodrive aborts it will not automatically stop the vehicle. Stopping
115  * may be undesirable since nearby enemies could get into the vehicle. It is up to the player
116  * to either stop or continue driving manually for a while. However, when aborting due to a
117  * collision (which happened despite collision detection, i.e. there was a simulation discrepancy
118  * near an obstacle) the brakes will be engaged.
119  *
120  * In order to prevent exploiting the system to navigate in the dark, autodrive will also
121  * perform visibility checks. Not being able to see anything in front of the vehicle will
122  * immediately cancel (or fail to start) autodrive. If the driver has only partial visibility
123  * of the front of the vehicle, safe mode will engage limiting the speed. Furthermore, the
124  * driver needs to either see or have map memory for the map tiles in the current OMT; all
125  * unseen tiles will be treated as obstacles during pathfinding. In practice, this means
126  * that driving at night should only be possible through familiar terrain with limited
127  * lights / night vision or through unfamiliar terrain but with good headlights / excellent
128  * night vision.
129  *
130  * In order to keep the memory footprint small and constant, this implementation uses a different
131  * coordinate system for the navigation caches than on the game map. The "nav map" is always
132  * allocated to the size of two OMTs wide and one OMT high. Picture it like having one OMT `a` on
133  * the left and one OMT `b` on the right. The coordinates of the nav map are translated and
134  * rotated so that `a` is always the current OMT and `b` is the next OMT we are trying to reach.
135  * So even if, for example, we are planning a route south, the nav map is still allocated as two
136  * OMTs wide, but its coordinates are instead rotated when we want to deal with actual game map
137  * coordinates.
138  */

633 vehicle_profile vehicle::autodrive_controller::compute_profile( orientation facing ) const
634 {
635  vehicle_profile ret;
636  tileray tdir( to_angle( facing ) );
637  ret.tdir = tdir;
638  std::map<int, std::pair<int, int>> extent_map;
639  const point pivot = driven_veh.pivot_point();
640  for( const vehicle_part &part : driven_veh.parts ) {
641  if( part.removed ) {
642  continue;
643  }
644  tripoint pos;
645  driven_veh.coord_translate( tdir, pivot, part.mount, pos );
646  if( extent_map.find( pos.y ) == extent_map.end() ) {
647  extent_map[pos.y] = { pos.x, pos.x };
648  } else {
649  auto &extent = extent_map[pos.y];
650  extent.first = std::min( extent.first, pos.x );
651  extent.second = std::max( extent.second, pos.x );
652  }
653  }
654  for( const auto &extent : extent_map ) {
655  const int y = extent.first;
656  const int x_min = extent.second.first;
657  const int x_max = extent.second.second;
658  for( int x = x_min; x <= x_max; x++ ) {
659  ret.occupied_zone.emplace_back( x, y );
660  }
661  }
662  for( int part_num : driven_veh.rotors ) {
663  const vehicle_part &part = driven_veh.part( part_num );
664  const int diameter = part.info().rotor_info->rotor_diameter;
665  const int radius = ( diameter + 1 ) / 2;
666  if( radius > 0 ) {
667  tripoint pos;
668  driven_veh.coord_translate( tdir, pivot, part.mount, pos );
669  for( tripoint pt : points_in_radius( pos, radius ) ) {
670  ret.occupied_zone.emplace_back( pt.xy() );
671  }
672  }
673  }
674  // figure out the maximum amount of displacement that can happen from advancing the
675  // tileray a single step by advancing it a bunch of times and reducing the x and y
676  // components of the displacement to at most 1
677  constexpr int far_advance = 100;
678  tdir.advance( far_advance );
679  const point increment( signum( tdir.dx() ), signum( tdir.dy() ) );
680  // compute the subset of vehicle points that would move to previously unoccupied spots
681  // (a.k.a. "moving first") when the vehicle moves one step; these are the only points
682  // we need to check for collision when the vehicle is moving in this direction
683  const std::unordered_set<point> occupied_set( ret.occupied_zone.begin(), ret.occupied_zone.end() );
684  for( const point &pt : ret.occupied_zone ) {
685  if( occupied_set.find( pt + increment ) == occupied_set.end() ) {
686  ret.collision_points.emplace_back( pt );
687  }
688  }
689  return ret;
690 }

692 // Return true if the map tile at the given position (in map coordinates)
693 // can be driven on (not an obstacle).
694 // The logic should match what is in vehicle::part_collision().
695 bool vehicle::autodrive_controller::check_drivable( const tripoint_bub_ms &pt ) const
696 {
697  const map &here = get_map();
698 
699  // check if another vehicle is there; tiles occupied by the current
700  // vehicle are evidently drivable
701  const optional_vpart_position ovp = here.veh_at( pt );
702  if( ovp ) {
703  // Known corner case: some furniture can be driven over, but will collide with
704  // wheel parts; if the vehicle starts over such a furniture we'll mark that tile
705  // as safe and may collide with it by turning; however if we mark it unsafe
706  // we'll have no viable paths away from the starting point.
707  return &ovp->vehicle() == &driven_veh;
708  }
709 
710  const tripoint_abs_ms pt_abs = here.getglobal( pt );
711  const tripoint_abs_omt pt_omt = project_to<coords::omt>( pt_abs );
712  // only check visibility for the current OMT, we'll check other OMTs when
713  // we reach them
714  if( pt_omt == data.current_omt ) {
715  // driver must see the tile or have seen it before in order to plan a route over it
716  if( !driver.sees( pt ) ) {
717  if( !driver.is_avatar() ) {
718  return false;
719  }
720  const avatar &avatar = *driver.as_avatar();
721  if( !avatar.is_map_memory_valid() ) {
722  debugmsg( "autodrive querying uninitialized map memory at %s", pt_abs.to_string() );
723  return false;
724  }
725  if( avatar.get_memorized_tile( pt_abs ) == mm_submap::default_tile ) {
726  // apparently open air doesn't get memorized, so pretend it is or else
727  // we can't fly helicopters due to the many unseen tiles behind the driver
728  if( !( data.air_ok && here.ter( pt ) == ter_t_open_air ) ) {
729  return false;
730  }
731  }
732  }
733  }
734 
735  // check for creatures
736  // TODO: padding around monsters
737  Creature *critter = get_creature_tracker().creature_at( pt, true );
738  if( critter && driver.sees( *critter ) ) {
739  return false;
740  }
741 
742  // don't drive over visible traps
743  if( here.can_see_trap_at( pt.raw(), driver ) ) {
744  return false;
745  }
746 
747  // check for furniture that hinders movement; furniture with 0 move cost
748  // can be driven on
749  const furn_id furniture = here.furn( pt );
750  if( furniture != furn_str_id::NULL_ID() && furniture.obj().movecost != 0 ) {
751  return false;
752  }
753 
754  const ter_id terrain = here.ter( pt );
755  if( terrain == ter_str_id::NULL_ID() ) {
756  return false;
757  }
758  // open air is an obstacle to non-flying vehicles; it is drivable
759  // for flying vehicles
760  if( terrain == ter_t_open_air ) {
761  return data.air_ok;
762  }
763  const ter_t &terrain_type = terrain.obj();
764  // watercraft can drive on water
765  if( data.water_ok && terrain_type.has_flag( ter_furn_flag::TFLAG_SWIMMABLE ) ) {
766  return true;
767  }
768  // remaining checks are for land-based navigation
769  if( !data.land_ok ) {
770  return false;
771  }
772  // NOLINTNEXTLINE(bugprone-branch-clone)
773  if( terrain_type.movecost <= 0 ) {
774  // walls and other impassable terrain
775  return false;
776  } else if( terrain_type.movecost == 2 || terrain_type.has_flag( ter_furn_flag::TFLAG_NOCOLLIDE ) ) {
777  // terrain with neutral move cost or tagged with NOCOLLIDE will never cause
778  // collisions
779  return true;
780  } else if( terrain_type.bash.str_max >= 0 && !terrain_type.bash.bash_below ) {
781  // bashable terrain (but not bashable floors) will cause collisions
782  return false;
783  } else if( terrain_type.has_flag( ter_furn_flag::TFLAG_LIQUID ) ) {
784  // water and lava
785  return false;
786  }
787  return true;
788 }

791 {

807 // Checks whether `p` is a drivable ramp up or down,
808 // and if so adds the ramp's destination tripoint to `ramp_points`
809 void vehicle::autodrive_controller::enqueue_if_ramp( point_queue &ramp_points,
810  const map &here, const tripoint_bub_ms &p ) const
811 {
812  if( !data.land_ok ) {
813  return;
814  }
815  ramp_points.visited.emplace( p );
816  if( p.z() < OVERMAP_HEIGHT && here.has_flag( ter_furn_flag::TFLAG_RAMP_UP, p ) ) {
817  ramp_points.enqueue( p + tripoint_above );
818  }
819  if( p.z() > -OVERMAP_DEPTH && here.has_flag( ter_furn_flag::TFLAG_RAMP_DOWN, p ) ) {
820  ramp_points.enqueue( p + tripoint_below );
821  }
822 }
823 
824 // Flood-fills from all enqueued points of up or down ramps. For each connected point, we
825 // set `is_obstacle` and `ground_z` again, based on whether they are an obstacle or not on
826 // this zlevel.
827 void vehicle::autodrive_controller::compute_obstacles_from_enqueued_ramp_points(
828  point_queue &ramp_points, const map &here )
829 {
830  auto is_drivable = [this, &here]( const tripoint_bub_ms & p ) {
831  return here.inbounds( p ) && check_drivable( p );
832  };
833  while( !ramp_points.to_check.empty() ) {
834  const tripoint_bub_ms ramp_point = ramp_points.to_check.front();
835  ramp_points.to_check.pop();
836  for( const tripoint_bub_ms &p : ff::point_flood_fill_4_connected( ramp_point, ramp_points.visited,
837  is_drivable ) ) {
838  const point pt_view = data.to_view( p );
839  if( !data.view_bounds.contains( pt_view ) ) {
840  continue;
841  }
842  // We now know that point p is drivable (and not an obstacle) on this zlevel, since
843  // it passed the is_drivable check.
844  if( data.is_obstacle[pt_view] ) {
845  // We have examined this point previously on a different zlevel but conluded
846  // that it was an obstacle at that zlevel. But on this zlevel it is apparently
847  // not an obstacle, so this must be our groundlevel.
848  data.ground_z[pt_view] = p.z();
849  } else {
850  data.ground_z[pt_view] = std::min( data.ground_z[pt_view], p.z() );
851  }
852  data.is_obstacle[pt_view] = false;
853  enqueue_if_ramp( ramp_points, here, p );
854  }
855  };
856 }

881 void vehicle::autodrive_controller::compute_goal_zone()
882 {
883  data.goal_zone.clear();
884  coord_transformation goal_transform;
885  if( data.next_next_omt.xy() != data.next_omt.xy() ) {
886  // set the goal at the edge of next_omt and next_next_omt (in next_omt
887  // space, pointing towards next_next_omt)
888  const point next_omt_middle( OMT_SIZE + OMT_SIZE / 2, OMT_SIZE / 2 );
889  const tripoint_rel_omt omt_diff = data.next_next_omt - data.next_omt;
890  const quad_rotation rotation = to_quad_rotation( omt_diff.raw().xy() ) - data.nav_to_map.rotation;
891  goal_transform = {next_omt_middle, rotation, next_omt_middle};
892  } else {
893  // set the goal at the edge of cur_omt and next_omt (in next_omt space,
894  // pointing away from cur_omt)
895  goal_transform = { point( OMT_SIZE - 1, 0 ), quad_rotation::d0, point_zero };
896  }
897  constexpr int max_turns = NUM_ORIENTATIONS / 8 + 1;
898  const int x = 2 * OMT_SIZE - 1;
899  for( int turns = -max_turns; turns <= max_turns; turns++ ) {
900  const orientation dir = orientation::d0 + turns;
901  static_assert( NAV_MAP_SIZE_Y == OMT_SIZE, "Unexpected nav map size" );
902  for( int y = 0; y < OMT_SIZE; y++ ) {
903  const point pt( x, y );
904  const node_address addr = goal_transform.transform( pt, dir );
905  if( data.valid_position( addr ) ) {
906  data.goal_zone.insert( addr );
907  }
908  }
909  }
910  data.goal_points[0] = point( OMT_SIZE, OMT_SIZE / 2 - 1 );
911  data.goal_points[1] = goal_transform.transform( point( x, OMT_SIZE / 2 - 1 ) );
912 }

964 scored_address vehicle::autodrive_controller::compute_node_score( const node_address &addr,
965  const navigation_node &node ) const
966 {
967  // TODO: tweak this
968  constexpr int cost_mult = 1;
969  constexpr int forward_dist_mult = 10;
970  constexpr int side_dist_mult = 8;
971  constexpr int angle_mult = 2;
972  constexpr int nearness_penalty = 15;
973  scored_address ret{ addr, 0 };
974  ret.score += cost_mult * node.cost;
975  if( node.is_goal ) {
976  return ret;
977  }
978  static constexpr std::array<point, 4> neighbor_deltas = {
979  point_east, point_south, point_west, point_north
980  };
981  for( const point &neighbor_delta : neighbor_deltas ) {
982  const point p = addr.get_point() + neighbor_delta;
983  if( !data.nav_bounds.contains( p ) || !data.valid_position( addr.facing_dir, p ) ) {
984  ret.score += nearness_penalty;
985  }
986  }
987  const int omt_index = std::min( std::max( addr.x / OMT_SIZE, 0 ),
988  static_cast<int>( data.goal_points.size() ) - 1 );
989  const point waypoint_delta = data.goal_points[omt_index] - addr.get_point();
990  const point goal_delta = data.goal_points.back() - addr.get_point();
991  const orientation waypoint_dir = approx_orientation( waypoint_delta.x, waypoint_delta.y );
992  ret.score += forward_dist_mult * std::abs( goal_delta.x );
993  ret.score += side_dist_mult * std::abs( goal_delta.y );
994  if( waypoint_delta.x > OMT_SIZE / 4 ) {
995  ret.score += angle_mult * std::abs( orientation_diff( addr.facing_dir, waypoint_dir ) );
996  }
997  return ret;

1000 void vehicle::autodrive_controller::compute_next_nodes( const node_address &addr,
1001  const navigation_node &node, int target_speed_tps,
1002  std::vector<std::pair<node_address, navigation_node>> &next_nodes )
1003 const
1004 {
1005  constexpr int move_cost = 10;
1006  constexpr int steering_cost = 1;
1007  const int sign = target_speed_tps > 0 ? 1 : -1;
1008  const int target_speed = target_speed_tps * VMIPH_PER_TPS;
1009  const int cur_omt = addr.x / OMT_SIZE;
1010  int next_speed = target_speed;
1011  int num_tiles_to_move = std::abs( target_speed_tps );
1012  if( target_speed_tps > 1 && node.speed < target_speed ) {
1013  const int cur_tps = std::min( std::max( node.speed / VMIPH_PER_TPS, 0 ), data.max_speed_tps - 1 );
1014  next_speed = std::min( std::max<int>( node.speed, 0 ) + data.acceleration[cur_tps], target_speed );
1015  num_tiles_to_move = next_speed / VMIPH_PER_TPS;
1016  }
1017  for( int steer = -data.max_steer; steer <= data.max_steer; steer++ ) {
1018  node_address next_addr = addr;
1019  next_addr.facing_dir = addr.facing_dir + steer;
1020  tileray tdir = data.profile( next_addr.facing_dir ).tdir;
1021  if( steer == 0 ) {
1022  tdir.advance( node.tileray_steps );
1023  }
1024  bool ok = true;
1025  bool goal_found = false;
1026  for( int i = 0; i < num_tiles_to_move && ok; i++ ) {
1027  tdir.advance( sign );
1028  next_addr.x += tdir.dx();
1029  next_addr.y += tdir.dy();
1030  if( !data.nav_bounds.contains( next_addr.get_point() ) ) {
1031  // it's ok to go out of bounds only from the "next" omt and only
1032  // after crossing the goal zone
1033  if( cur_omt == 0 || !goal_found ) {
1034  ok = false;
1035  }
1036  } else if( !data.valid_position( next_addr ) ) {
1037  ok = false;
1038  } else if( !goal_found && data.goal_zone.find( next_addr ) != data.goal_zone.end() ) {
1039  goal_found = true;
1040  }
1041  }
1042  if( !ok ) {
1043  continue;
1044  }
1045  const int next_omt = next_addr.x / OMT_SIZE;
1046  if( next_omt < cur_omt ) {
1047  // no going back to a previous omt
1048  continue;
1049  }
1050  navigation_node next_node;
1051  next_node.prev = addr;
1052  next_node.tileray_steps = num_tiles_to_move + ( !steer ? node.tileray_steps : 0 );
1053  // TODO: improve this
1054  next_node.cost = node.cost + move_cost * num_tiles_to_move + ( steer ? steering_cost : 0 );
1055  next_node.speed = next_speed;
1056  next_node.target_speed_tps = target_speed_tps;
1057  next_node.is_goal = goal_found;
1058  next_nodes.emplace_back( next_addr, next_node );
1059  }
1060 }

1062 std::optional<std::vector<navigation_step>> vehicle::autodrive_controller::compute_path(
1063  int speed_tps ) const
1064 {
1065  if( speed_tps == 0 || speed_tps < -1 ) {
1066  return std::nullopt;
1067  }
1068  // TODO: tweak this
1069  constexpr int max_search_count = 10000;
1070  std::vector<navigation_step> ret;
1071  // TODO: check simple reachability first and bail out or set upper bound on node score
1072  std::unordered_map<node_address, navigation_node, node_address_hasher> known_nodes;
1073  std::priority_queue<scored_address, std::vector<scored_address>, std::greater<>>
1074  open_set;
1075  const tripoint_abs_ms veh_pos = driven_veh.global_square_location();
1076  const node_address start = data.nav_to_map.inverse().transform(
1077  veh_pos.raw().xy(), to_orientation( driven_veh.face.dir() ) );
1078  known_nodes.emplace( start, make_start_node( start, driven_veh ) );
1079  open_set.push( scored_address{ start, 0 } );
1080  std::vector<std::pair<node_address, navigation_node>> next_nodes;
1081  while( !open_set.empty() ) {
1082  const node_address cur_addr = open_set.top().addr;
1083  open_set.pop();
1084  const navigation_node &cur_node = known_nodes[cur_addr];
1085  if( cur_node.is_goal ) {
1086  node_address addr = cur_addr;
1087  while( !( addr == start ) ) {
1088  const navigation_node &node = known_nodes.at( addr );
1089  const node_address &prev = node.prev;
1090  const tripoint_abs_ms prev_loc( data.nav_to_map.transform( prev.get_point(),
1091  data.current_omt.z() ) );
1092  ret.emplace_back( navigation_step{
1093  data.adjust_z( prev_loc ),
1094  data.nav_to_map.transform( addr.facing_dir ),
1095  node.target_speed_tps
1096  } );
1097  addr = prev;
1098  }
1099  return ret;
1100  }
1101  next_nodes.clear();
1102  compute_next_nodes( cur_addr, cur_node, speed_tps, next_nodes );
1103  for( const auto &next : next_nodes ) {
1104  const node_address &next_addr = next.first;
1105  const navigation_node &next_node = next.second;
1106  auto iter = known_nodes.find( next_addr );

1148  tileray face_dir = driven_veh.face;

1152  bool blind = true;
1153  for( const point &p : data.profile( to_orientation( face_dir.dir() ) ).collision_points ) {

1159  // up or down ramps, which makes this check think that we should stop pathfinding.
1160  // So here we allow having no visibility in case the path changes zlevel.
1161  // Checks further down below in this method will make sure that we instead slow down.
1162  changed_zlevel |= ( next.z() != veh_pos.z() );

1166  }

1169  // should force us to stop

1171  tileray tdir = driven_veh.face;
1172  if( turn_dir != to_orientation( tdir.dir() ) ) {
1173  tdir = profile.tdir;
1174  }
1175  const int speed = std::min( driven_veh.velocity + driven_veh.current_acceleration(),
1176  driven_veh.cruise_velocity );

1181  for( const point &p : profile.occupied_zone ) {
1182  collision_zone.insert( p + offset );

1187  }
1188  }
1189

1191  collision_zone.clear();

1193  tdir.advance();
1194  offset += point( tdir.dx(), tdir.dy() );
1195  for( const point &p : profile.collision_points ) {
1196  collision_zone.insert( p + offset );
1197  }
1198  }
1199  for( const point &p : collision_zone ) {
1200  const tripoint_bub_ms next = data.adjust_z( veh_pos + p );
1201  if( !driver.sees( next ) ) {
1202  return collision_check_result::slow_down;
1203  }
1204  if( !check_drivable( next ) ) {
1205  return collision_check_result::slow_down;
1206  }

1216 std::optional<navigation_step> vehicle::autodrive_controller::compute_next_step()
1217 {
1218  precompute_data();
1219  const tripoint_abs_ms veh_pos = driven_veh.global_square_location();
1220  const bool had_cached_path = !data.path.empty();
1221  const bool two_steps = data.path.size() > 2;
1222  const navigation_step first_step = two_steps ? data.path.back() : navigation_step();
1223  const navigation_step second_step = two_steps ? data.path.at( data.path.size() - 2 ) :
1224  navigation_step();
1225  bool maintain_speed = false;
1226  // If vehicle did not move as far as planned and direction is the same
1227  // then it is still accelerating. If the vehicle moved more than expected
1228  // then we likely underestimated the acceleration when planning the path.
1229  // If either of these happen, we should maintain speed but compute a new path.
1230  if( two_steps && square_dist( first_step.pos.xy().raw(), second_step.pos.xy().raw() ) !=
1231  square_dist( first_step.pos.xy().raw(), veh_pos.xy().raw() ) &&
1232  first_step.steering_dir == second_step.steering_dir ) {
1233  data.path.pop_back();
1234  maintain_speed = true;
1235  data.path.clear();
1236  } else {
1237  while( !data.path.empty() && data.path.back().pos != veh_pos ) {
1238  data.path.pop_back();
1239  }
1240  }
1241  if( !data.path.empty() && data.path.back().target_speed_tps > data.max_speed_tps ) {
1242  data.path.clear();
1243  maintain_speed = false;
1244  }
1245  if( data.path.empty() ) {
1246  // if we're just starting out or we've gone off-course use the lowest speed
1247  if( ( had_cached_path && !maintain_speed ) || driven_veh.velocity == 0 ) {
1248  data.max_speed_tps = MIN_SPEED_TPS;
1249  }
1250  auto new_path = compute_path( data.max_speed_tps );
1251  while( !new_path && data.max_speed_tps > MIN_SPEED_TPS ) {
1252  // high speed didn't work, try a lower speed
1253  data.max_speed_tps /= 2;
1254  new_path = compute_path( data.max_speed_tps );
1255  }
1256  if( !new_path ) {
1257  return std::nullopt;
1258  }
1259  data.path.swap( *new_path );
1260  }
1261  return data.path.back();
1262 }

1376  std::optional<navigation_step> next_step = active_autodrive_controller->compute_next_step();

1379  driver.add_msg_if_player( _( "Can't see a path forward." ) );
1380  stop_autodriving( false );
1381  return autodrive_result::abort;
1382  }
1383  if( next_step->pos.xy() != veh_pos.xy() ) {
1384  debugmsg( "compute_next_step returned an invalid result" );
1385  stop_autodriving();
1386  return autodrive_result::abort;
1387  }
1388  if( next_step->target_speed_tps == 0 && velocity == 0 ) {
1389  stop_autodriving( false );
1390  return autodrive_result::finished;
1391  }
1392  cruise_velocity = next_step->target_speed_tps * VMIPH_PER_TPS;
1393 
1394  // check for collisions before we steer, since steering may end our turn
1395  // which would cause the vehicle to move and maybe crash
1396  switch( active_autodrive_controller->check_collision_zone( next_step->steering_dir ) ) {
1397  case collision_check_result::no_visibility:
1398  driver.add_msg_if_player( m_warning, _( "You can't see anything in front of your vehicle!" ) );

1401  case collision_check_result::close_obstacle:

1407  if( cruise_velocity > VMIPH_PER_TPS ) {
1408  cruise_velocity = VMIPH_PER_TPS;
1409  }
1410  break;
1411  case collision_check_result::ok:
1412  break;
1413  }
1414 
1415  int turn_delta = orientation_diff( next_step->steering_dir, to_orientation( turn_dir ) );
1416  // pldrive() does not handle steering multiple times in one call correctly
1417  // call it multiple times, matching how a player controls the vehicle
1418  for( int i = 0; i < std::abs( turn_delta ); i++ ) {
1419  if( driver.get_moves() <= 0 ) {
1420  // we couldn't steer as many times as we wanted to but there's
1421  // nothing we can do about it now, hope we don't crash!
1422  break;
1423  }
1424  pldrive( driver, { signum( turn_delta ), 0 } );
1425  }
1426  // Don't do anything else below; the driver's turn may be over (moves <= 0) so
1427  // any extra actions would be "cheating".
1428
